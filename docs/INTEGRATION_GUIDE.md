# Integration Guide

This guide walks you through integrating with the Vestra token distribution program from scratch. By the end you will have a working TypeScript script that creates a linear vesting stream, waits for tokens to vest, and withdraws them.

**Program ID:** `J4zBUJeaXA26nV6i9Jz45t4hfwNrsxZ96g5ozhwALfX3`  
**Network:** Solana Devnet  
**Anchor version:** 0.32.1

---

## Prerequisites

- Node.js 20 or later
- A Solana wallet with devnet SOL (get some with `solana airdrop 2`)
- Basic familiarity with Solana accounts and public keys

## Step 2: Load the IDL and create the program client

The IDL (Interface Definition Language file) describes every instruction, account, and type in the program. It is generated by the Anchor build and lives at `contracts/target/idl/token_distribution_protocol.json`.

```typescript
import * as anchor from "@coral-xyz/anchor";
import { AnchorProvider, Program, BN } from "@coral-xyz/anchor";
import { Connection, Keypair, PublicKey } from "@solana/web3.js";
import {
  getAssociatedTokenAddress,
  getOrCreateAssociatedTokenAccount,
  TOKEN_PROGRAM_ID,
  ASSOCIATED_TOKEN_PROGRAM_ID,
} from "@solana/spl-token";
import * as fs from "fs";

// Load your wallet keypair
const walletKeypair = Keypair.fromSecretKey(
  Uint8Array.from(JSON.parse(fs.readFileSync(process.env.WALLET_PATH!, "utf-8")))
);

// Connect to devnet
const connection = new Connection("https://api.devnet.solana.com", "confirmed");

// Build a NodeWallet and Provider
const wallet = new anchor.Wallet(walletKeypair);
const provider = new AnchorProvider(connection, wallet, {
  commitment: "confirmed",
});
anchor.setProvider(provider);

// Load the IDL and create a typed program client
const idl = JSON.parse(
  fs.readFileSync("./contracts/target/idl/token_distribution_protocol.json", "utf-8")
);
const programId = new PublicKey("J4zBUJeaXA26nV6i9Jz45t4hfwNrsxZ96g5ozhwALfX3");
const program = new Program(idl, programId, provider);
```

## Step 3: Get test tokens

The program ships with a `mint_mock_tokens` instruction that gives you up to 1,000,000 test tokens on devnet.

```typescript
async function getMockTokens(amount: number): Promise<PublicKey> {
  // Derive the program-controlled mock mint
  const [mockMintPda] = PublicKey.findProgramAddressSync(
    [Buffer.from("mock_mint")],
    program.programId
  );

  // Get or create your token account for this mint
  const minterAta = await getAssociatedTokenAddress(
    mockMintPda,
    walletKeypair.publicKey
  );

  await program.methods
    .mintMockTokens(new BN(amount))
    .accounts({
      minter: walletKeypair.publicKey,
      mockMint: mockMintPda,
      minterTokenAccount: minterAta,
      tokenProgram: TOKEN_PROGRAM_ID,
      associatedTokenProgram: ASSOCIATED_TOKEN_PROGRAM_ID,
      systemProgram: anchor.web3.SystemProgram.programId,
    })
    .rpc();

  console.log(`Minted ${amount} mock tokens to ${minterAta.toBase58()}`);
  return mockMintPda;
}
```

## Step 4: Derive the PDA addresses

Two accounts are created per stream. Their addresses are deterministic and can be computed entirely off-chain.

```typescript
function deriveStreamPdas(
  creator: PublicKey,
  recipient: PublicKey,
  streamId: BN,
  programId: PublicKey
): { streamDataPda: PublicKey; escrowPda: PublicKey } {
  const [streamDataPda] = PublicKey.findProgramAddressSync(
    [
      Buffer.from("stream"),
      creator.toBuffer(),
      recipient.toBuffer(),
      streamId.toArrayLike(Buffer, "le", 8),
    ],
    programId
  );

  const [escrowPda] = PublicKey.findProgramAddressSync(
    [Buffer.from("escrow"), streamDataPda.toBuffer()],
    programId
  );

  return { streamDataPda, escrowPda };
}
```

## Step 5: Create a linear vesting stream

This creates a stream that vests tokens linearly over one hour. We use `Date.now()` as the stream ID to guarantee uniqueness.

```typescript
async function createLinearStream(
  mint: PublicKey,
  recipient: Keypair,
  totalAmount: number
): Promise<{ streamDataPda: PublicKey; escrowPda: PublicKey; streamId: BN }> {
  const streamId = new BN(Date.now());
  const now = Math.floor(Date.now() / 1000);

  const { streamDataPda, escrowPda } = deriveStreamPdas(
    walletKeypair.publicKey,
    recipient.publicKey,
    streamId,
    program.programId
  );

  const creatorAta = await getAssociatedTokenAddress(
    mint,
    walletKeypair.publicKey
  );

  await program.methods
    .createStream(
      streamId,
      new BN(totalAmount),          // amount to lock
      new BN(now),                  // start_time: now
      new BN(now),                  // cliff_time: same as start (no cliff)
      new BN(now + 3600),           // end_time: 1 hour from now
      0,                            // stream_type: 0 = Linear
      true                          // is_cancelable
    )
    .accounts({
      creator: walletKeypair.publicKey,
      recipient: recipient.publicKey,
      streamData: streamDataPda,
      escrowTokenAccount: escrowPda,
      creatorTokenAccount: creatorAta,
      mint,
      tokenProgram: TOKEN_PROGRAM_ID,
      associatedTokenProgram: ASSOCIATED_TOKEN_PROGRAM_ID,
      systemProgram: anchor.web3.SystemProgram.programId,
    })
    .signers([walletKeypair])
    .rpc();

  console.log(`Stream created. PDA: ${streamDataPda.toBase58()}`);
  console.log(`Escrow: ${escrowPda.toBase58()}`);

  return { streamDataPda, escrowPda, streamId };
}
```

## Step 6: Read stream state

Fetch and print the current state of a stream at any time.

```typescript
async function readStream(streamDataPda: PublicKey): Promise<void> {
  const stream = await program.account.streamData.fetch(streamDataPda);

  const now = Math.floor(Date.now() / 1000);
  const elapsed = now - stream.startTime.toNumber();
  const duration = stream.endTime.toNumber() - stream.startTime.toNumber();
  const percentVested = Math.min((elapsed / duration) * 100, 100).toFixed(1);

  console.log("--- Stream State ---");
  console.log(`Creator:        ${stream.creator.toBase58()}`);
  console.log(`Recipient:      ${stream.recipient.toBase58()}`);
  console.log(`Total:          ${stream.amountTotal.toNumber()}`);
  console.log(`Claimed:        ${stream.amountClaimed.toNumber()}`);
  console.log(`Vested:         ~${percentVested}%`);
  console.log(`Cancelled:      ${stream.isCancelled}`);
  console.log(`Type:           ${["Linear", "Cliff+Linear", "Milestone"][stream.streamType]}`);
}
```

## Step 7: Withdraw vested tokens

The recipient calls this to claim whatever has vested since the last withdrawal.

```typescript
async function withdraw(
  recipient: Keypair,
  mint: PublicKey,
  streamDataPda: PublicKey,
  escrowPda: PublicKey,
  creatorPubkey: PublicKey
): Promise<void> {
  const recipientAta = await getAssociatedTokenAddress(
    mint,
    recipient.publicKey
  );

  await program.methods
    .withdraw()
    .accounts({
      recipient: recipient.publicKey,
      creator: creatorPubkey,
      streamData: streamDataPda,
      escrowTokenAccount: escrowPda,
      recipientTokenAccount: recipientAta,
      mint,
      tokenProgram: TOKEN_PROGRAM_ID,
      associatedTokenProgram: ASSOCIATED_TOKEN_PROGRAM_ID,
      systemProgram: anchor.web3.SystemProgram.programId,
    })
    .signers([recipient])
    .rpc();

  console.log("Withdrawal successful.");
}
```

## Step 8: Cancel a stream

Only the creator can call this, and only if `is_cancelable` was `true` at creation time. Vested tokens go to the recipient; unvested tokens return to the creator.

```typescript
async function cancelStream(
  recipient: PublicKey,
  mint: PublicKey,
  streamDataPda: PublicKey,
  escrowPda: PublicKey
): Promise<void> {
  const creatorAta = await getAssociatedTokenAddress(
    mint,
    walletKeypair.publicKey
  );
  const recipientAta = await getAssociatedTokenAddress(mint, recipient);

  await program.methods
    .cancel()
    .accounts({
      creator: walletKeypair.publicKey,
      recipient,
      streamData: streamDataPda,
      escrowTokenAccount: escrowPda,
      recipientTokenAccount: recipientAta,
      creatorTokenAccount: creatorAta,
      mint,
      tokenProgram: TOKEN_PROGRAM_ID,
      associatedTokenProgram: ASSOCIATED_TOKEN_PROGRAM_ID,
      systemProgram: anchor.web3.SystemProgram.programId,
    })
    .signers([walletKeypair])
    .rpc();

  console.log("Stream cancelled.");
}
```

## Step 9: Handle errors

Wrap calls in a try/catch and inspect the Anchor error code for clean error handling.

```typescript
import { AnchorError } from "@coral-xyz/anchor";

async function safeWithdraw(...args: Parameters<typeof withdraw>) {
  try {
    await withdraw(...args);
  } catch (err) {
    if (err instanceof AnchorError) {
      const code = err.error.errorCode.number;
      const name = err.error.errorCode.code;
      console.error(`Program error ${code}: ${name}`);

      if (code === 6009) {
        console.log("No tokens have vested yet. Try again later.");
      } else if (code === 6014) {
        console.log("This stream has been cancelled.");
      }
    } else {
      throw err;
    }
  }
}
```

## Full example: linear stream end-to-end

```typescript
async function main() {
  // 1. Fund the wallet with test tokens
  const mint = await getMockTokens(100_000);

  // 2. Create a recipient keypair
  const recipient = Keypair.generate();

  // 3. Airdrop some SOL to the recipient so they can pay for transactions
  const sig = await connection.requestAirdrop(recipient.publicKey, 1e9);
  await connection.confirmTransaction(sig);

  // 4. Create a stream: lock 100,000 tokens vesting over 1 hour
  const { streamDataPda, escrowPda } = await createLinearStream(
    mint,
    recipient,
    100_000
  );

  // 5. Print the stream state immediately (0% vested)
  await readStream(streamDataPda);

  // 6. Wait 30 seconds so ~0.83% vests (30s / 3600s)
  //    In a real scenario you would wait longer or use a test validator with
  //    time manipulation.
  console.log("Waiting 30 seconds...");
  await new Promise((r) => setTimeout(r, 30_000));

  // 7. Withdraw whatever has vested
  await safeWithdraw(
    recipient,
    mint,
    streamDataPda,
    escrowPda,
    walletKeypair.publicKey
  );

  // 8. Print updated stream state
  await readStream(streamDataPda);
}

main().catch(console.error);
```

## Creating a milestone stream

Milestone streams follow a slightly different setup: create the stream with `stream_type = 2`, then add milestones before any withdrawal occurs.

```typescript
async function createMilestoneStream(
  mint: PublicKey,
  recipient: Keypair
): Promise<{ streamDataPda: PublicKey; escrowPda: PublicKey }> {
  const streamId = new BN(Date.now());
  const now = Math.floor(Date.now() / 1000);

  const { streamDataPda, escrowPda } = deriveStreamPdas(
    walletKeypair.publicKey,
    recipient.publicKey,
    streamId,
    program.programId
  );

  const creatorAta = await getAssociatedTokenAddress(
    mint,
    walletKeypair.publicKey
  );

  // Create stream with type 2 (Milestone). Time fields are required by the
  // instruction but not used for milestone vesting calculation.
  await program.methods
    .createStream(
      streamId,
      new BN(1_000_000),
      new BN(now),
      new BN(now),
      new BN(now + 365 * 24 * 3600),
      2,      // stream_type: 2 = Milestone
      false   // is_cancelable: false (payment guarantee)
    )
    .accounts({
      creator: walletKeypair.publicKey,
      recipient: recipient.publicKey,
      streamData: streamDataPda,
      escrowTokenAccount: escrowPda,
      creatorTokenAccount: creatorAta,
      mint,
      tokenProgram: TOKEN_PROGRAM_ID,
      associatedTokenProgram: ASSOCIATED_TOKEN_PROGRAM_ID,
      systemProgram: anchor.web3.SystemProgram.programId,
    })
    .rpc();

  // Add milestone 1: deliver prototype (unlocks 400,000 tokens)
  const verifier = walletKeypair; // in practice this would be a separate oracle keypair
  const hash1 = Array.from(
    require("crypto").createHash("sha256").update("Deliver prototype by June 30").digest()
  );

  await program.methods
    .addMilestone(new BN(400_000), hash1, verifier.publicKey)
    .accounts({
      creator: walletKeypair.publicKey,
      recipient: recipient.publicKey,
      streamData: streamDataPda,
      systemProgram: anchor.web3.SystemProgram.programId,
    })
    .rpc();

  // Add milestone 2: launch mainnet (unlocks 600,000 tokens)
  const hash2 = Array.from(
    require("crypto").createHash("sha256").update("Launch on mainnet").digest()
  );

  await program.methods
    .addMilestone(new BN(600_000), hash2, verifier.publicKey)
    .accounts({
      creator: walletKeypair.publicKey,
      recipient: recipient.publicKey,
      streamData: streamDataPda,
      systemProgram: anchor.web3.SystemProgram.programId,
    })
    .rpc();

  // When the first milestone is completed, the verifier calls verify_milestone
  await program.methods
    .verifyMilestone(0)   // milestone index 0
    .accounts({
      verifier: verifier.publicKey,
      creator: walletKeypair.publicKey,
      recipient: recipient.publicKey,
      streamData: streamDataPda,
      systemProgram: anchor.web3.SystemProgram.programId,
    })
    .rpc();

  console.log("Milestone 0 verified. Recipient can now withdraw 400,000 tokens.");

  return { streamDataPda, escrowPda };
}
```

## Querying all streams for a wallet

Use `getProgramAccounts` with a memcmp filter to fetch every stream where a given wallet is the creator.

```typescript
async function getStreamsForCreator(creator: PublicKey) {
  // The StreamData account layout has an 8-byte discriminator, then:
  //   creator: Pubkey at offset 8
  const accounts = await connection.getProgramAccounts(programId, {
    filters: [
      {
        memcmp: {
          offset: 8, // skip 8-byte Anchor discriminator
          bytes: creator.toBase58(),
        },
      },
    ],
  });

  return accounts.map((a) =>
    program.coder.accounts.decode("StreamData", a.account.data)
  );
}
```

## Common issues

| Symptom | Cause | Fix |
|---------|-------|-----|
| `NothingToWithdraw (6009)` | No tokens have vested yet | Wait for the vesting schedule to advance or verify a milestone first |
| `StreamExpired (6014)` | Calling withdraw on a cancelled stream | Check `stream.isCancelled` before calling withdraw |
| `Unauthorized (6007)` | Wrong wallet is signing | Make sure the recipient signs withdraw; creator signs cancel |
| `InsufficientFunds (6003)` | Creator ATA balance < amount | Mint more test tokens or reduce the stream amount |
| `Account not found` | PDA not derived correctly | Double-check seed order: `["stream", creator, recipient, stream_id_le8]` |
| `SelfVesting (6004)` | Creator and recipient are the same key | Use two different keypairs |
