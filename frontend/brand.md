## Tone and voice

### Words to use

Direct, specific, number-forward. State what something does, not what it feels like. Prefer active verbs: "lock", "release", "claim", "schedule". When describing a feature, lead with the outcome for the user, not the mechanism.

### Words to avoid

Hype words: revolutionary, game-changing, unleash, supercharge, seamless, powerful. Jargon that requires Web3 knowledge to parse: "trustless", "on-chain", "SPL", "mint address", "raw units". Exclamation marks outside genuine moments of success. "Unlock" is overloaded in this domain — use "release" or "claim" instead.

### Voice example

> "Your tokens release on schedule, automatically. No manual transfers, no spreadsheets, no missed payouts."

## Usage dos and don'ts

**Do:**

- Use shadcn CSS variables everywhere: `bg-background`, `text-foreground`, `bg-primary`. Never hardcode hex.
- Keep the 4px spacing grid: `gap-2`, `p-4`, `px-6`.
- Use `font-mono tabular-nums` for all numbers that update in place (balances, amounts, schedules).
- Test every component in both light and dark mode before marking done.
- Use `bg-brand-accent` on the single most important CTA per screen, not on every button.
- Replace Web3 jargon with plain equivalents before shipping any copy to production.

**Don't:**

- Hardcode colors in component files.
- Use `transition: all` — specify `transition-colors`, `transition-transform`.
- Stack both gradients on the same screen.
- Place body text directly on `--gradient-accent`.
- Override shadcn tokens per-component — fix it in `globals.css` once.
- Use "trustless", "protocol", "SPL", "mint", or "raw units" in UI-facing copy.
